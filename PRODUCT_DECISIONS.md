# Main Product Decisions

- The page is structured like a normal browse experience, with lightweight store chrome, a compact input area, and dense product cards rather than a feature-marketing layout.
- The prototype loads with three favorite books already selected so the recommendation value is visible immediately, while still allowing the user to remove titles and rebuild the list.
- Recommendations are restricted to currently available inventory states so the feature stays tied to purchasable catalog items instead of abstract taste matching.
- Each card uses a short recommendation reason instead of a long narrative explanation to keep the scan pattern close to a standard ecommerce product grid.
- The optional focus controls use shopper-friendly language such as `Similar vibe`, `Different author`, `Budget friendly`, and `Page turner` because they feel believable in a retail interface.
- A smaller inline module is included below the main experience to show how the same system could fit naturally on a book detail page without creating a separate app-like flow.
